# ML-Powered Firewall IPS

An adaptive Intrusion Prevention System that learns from network traffic to detect
and block attacks in real time, using machine learning over behavioral flow features
extracted from Linux `iptables` logs.

## What it does

The system works in three stages:

1. **Sense** — A Linux server runs `iptables` with a `LOG` rule that captures every
   incoming packet to `kern.log`. A Python collector parses these logs over SSH.
2. **Decide** — Packets are aggregated into per IP, time-windowed *flows* with around 20
   behavioral features (SYN ratios, port diversity, packet rate, protocol mix, etc).
   A trained model classifies each flow as benign or malicious.
3. **Act** — When a malicious flow is detected, the agent issues an `iptables` rule
   to block the source IP.

The model is retrained periodically on accumulated, ground-truth-labeled data to
improve over time without being vulnerable to adaptive attackers (no online learning).

## Why behavioral flows instead of per-packet ML?

A single packet has no context, port number alone isn't enough to tell legitimate
traffic from a port scan. By aggregating packets per source IP over a 60 second
window, we get features that describe *behavior*: "did this IP touch 1000 ports
in one minute?" or "did it complete its TCP handshakes or just SYN and bail?"
These features carry real signal. Per packet features don't.

## Lab setup

Three machines on a host-only VirtualBox network:

| Role     | OS           | IP                | Purpose                              |
| -------- | ------------ | ----------------- | ------------------------------------ |
| Server   | Ubuntu 24.04 | `192.168.56.102`  | Runs iptables LOG sensor, target     |
| Attacker | Kali Linux   | `192.168.56.103`  | Generates malicious traffic          |
| Operator | Windows      | host              | Runs Python orchestrator             |

## Detected attack types

The training dataset is generated by running known-malicious traffic from Kali with
recorded timestamps, then labeling matching flows as ground truth:

- **SYN scan** (`nmap -sS`) — classic TCP port scan
- **FIN scan** (`nmap -sF`) — stealth variant using FIN flags
- **UDP scan** (`nmap -sU`) — UDP-protocol port enumeration
- **Slow scan** (`nmap -T1`) — rate-evasion port scan
- **SYN flood** (`hping3 --flood -S`) — denial-of-service
- **SSH brute force** — repeated failed authentication attempts on port 22

Each attack stresses different features, forcing the model to learn
multi-dimensional decision rules rather than memorizing one signature.

## Running it locally

```bash
# 1. Clone and set up environment
git clone https://github.com/CezarMarinescu/ml-firewall-ips.git
cd ml-firewall-ips
python -m venv .venv
.\.venv\Scripts\activate     # PowerShell on Windows
pip install -r requirements.txt

# 2. Configure credentials
copy .env.example .env       # then edit .env with real values

# 3. On the Ubuntu server, enable firewall logging
sudo iptables -F
sudo iptables -A INPUT -j LOG --log-prefix "FW_LOG: "

# 4. Generate attack data and collect logs
python -m scripts.simulate_attacks
python -m scripts.collect_data
python -m scripts.relabel_with_manifest
```
